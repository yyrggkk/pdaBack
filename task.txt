Create a Laravel API endpoint GET /api/menu that returns all menu categories with their nested articles.

- Use Eloquent eager loading to optimize the query
- Create a MenuController with an index method
- Return proper JSON response with 200 status

### TASK MO-B2: API POST /api/commandes
**Priority:** CRITIQUE | **Phase:** 2.4

**Prompt:**
Create a Laravel API endpoint POST /api/commandes to create a new order from the cart.

Requirements:
- Route: POST /api/commandes
- Request payload structure:
  ```json
  {
    "table_id": 12,
    "couverts": 2,
    "lignes": [
      { "article_id": 1, "quantite": 2 },
      { "article_id": 5, "quantite": 1 }
    ]
  }
  ```
- Validate all input fields (table_id required, couverts required integer min:1, lignes required array)
- Calculate the total automatically by summing (article.prix * quantite) for each line
- Create entry in `commandes` table with: table_id, couverts, total, statut='en_cuisine', created_at
- Create entries in `ligne_commandes` table for each item: commande_id, article_id, quantite, prix_unitaire
- Update the table status to "occupee" in the `tables` table
- Return the created order with its ID and all details:
  ```json
  {
    "id": 2145,
    "table_id": 12,
    "couverts": 2,
    "total": 60.50,
    "statut": "en_cuisine",
    "lignes": [...],
    "created_at": "..."
  }
  ```
- Create Commande model with relationships (belongsTo Table, hasMany LigneCommande)
- Create LigneCommande model with relationships (belongsTo Commande, belongsTo Article)
- Create migrations for `commandes` table (id, table_id, couverts, total, statut, created_at, updated_at)
- Create migrations for `ligne_commandes` table (id, commande_id, article_id, quantite, prix_unitaire, created_at, updated_at)
- Use database transaction to ensure data integrity
- Create CommandeController with store method

### TASK MO-F4: Store Panier (Zustand)
**Priority:** CRITIQUE | **Phase:** 4.3

**Prompt:**
Create a Zustand store for managing the shopping cart state.

Requirements:
- File: stores/cartStore.ts
- State interface:
  ```typescript
  interface CartItem {
    article_id: number;
    nom: string;
    prix: number;
    quantite: number;
    image_url: string;
  }
  
  interface CartState {
    items: CartItem[];
    tableId: number | null;
    couverts: number;
    
    // Actions
    setTableId: (id: number) => void;
    setCouverts: (count: number) => void;
    addItem: (article: Omit<CartItem, 'quantite'>) => void;
    removeItem: (articleId: number) => void;
    updateQuantity: (articleId: number, quantite: number) => void;
    clearCart: () => void;
    
    // Computed (as getters or functions)
    getItemCount: () => number;
    getTotalPrice: () => number;
    getItemQuantity: (articleId: number) => number;
  }
  ```
- Actions behavior:
  - addItem: If item exists, increment quantity; else add with quantity 1
  - removeItem: Decrement quantity; if 0, remove from array
  - updateQuantity: Set exact quantity; if 0, remove item
  - clearCart: Reset items to [], keep tableId and couverts
- Computed:
  - getItemCount: Sum of all quantities
  - getTotalPrice: Sum of (prix * quantite) for all items
- Use Zustand with TypeScript properly typed

### TASK MO-F7: Envoi Commande
**Priority:** CRITIQUE | **Phase:** 4.3

**Prompt:**
Implement the order submission functionality when "Send to Kitchen" is pressed.

Requirements:
- File: Can be in CartScreen.tsx or separate hooks/useSubmitOrder.ts
- Flow:
  1. User taps "SEND TO KITCHEN" button
  2. Show loading state (spinner on button, disable interactions)
  3. POST to /api/commandes via axios with payload:
     ```json
     {
       "table_id": cartStore.tableId,
       "couverts": cartStore.couverts,
       "lignes": cartStore.items.map(item => ({
         "article_id": item.article_id,
         "quantite": item.quantite
       }))
     }
     ```
  4. Handle response:
     - Success: Clear cart (cartStore.clearCart()), show success toast/alert, navigate to Plan des Tables
     - Error: Show error message, keep cart intact, allow retry
- Error handling:
  - Network error: "Connexion impossible. Vérifiez votre réseau."
  - Server error (500): "Erreur serveur. Réessayez."
  - Validation error (422): Display specific error messages
- UX:
  - Prevent double-submission
  - Show loading indicator during request
  - Confirm navigation away if cart has items (optional)
- Use React Navigation for navigation after success
